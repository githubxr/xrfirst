

# 30 Day Plan

（极简可执行版）

### 🎯 总目标

做一个可跑、可展示、可复用的「商品 + 订单 + 库存 + 日志」业务链路，并能应对面试。

包含：
- 后端微服务结构完整
- Redis 缓存、锁
- 事务传播
- 下单 → 检查库存 → 扣库存 → 写订单 → 写日志
- README 能让面试官秒懂



 阶段划分

### 🧩阶段0：查漏补缺（并行）
多线程、设计模式、前端 watch/ref 等内容，按需补。



### 🧩阶段1：微服务结构搭建
- api / busi / start 分层
- nacos、feign、网关
- feign fallback、统一异常



### 🧩阶段2：SQL训练
- 窗口函数
- 多表 join
- 聚合
- 排名/比较/环比



### 🧩阶段3：事务 + 隔离级别 + 锁
- 本地事务
- 事务传播
- 悲观锁、乐观锁
- 超卖/少卖实验
- **Redis**：

  1. 双写/失效一致性实现
  2. 热点 key / 缓存穿透、击穿
  3. 流程图 + Markdown 流程说明

  **产出**：SQL 实例 + Redis 缓存实验完成，带可复用代码



### 🧩阶段4：完整业务链路
- 链路1：下单
- 链路2：模拟支付
- 业务幂等
- 分布式锁
- Redis 缓存穿透/击穿/雪崩
- 简易 SAGA（补偿版）





### 🎁 最终产出

- README（自解释）
- 架构图（简单）
- 完整链路截图
- 可展示 demo
- 关键 SQL 示例（截图）
- 锁/缓存/事务的面试话术模板.md
